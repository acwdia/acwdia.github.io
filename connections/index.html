Introduction

Web Development is a combination of science and art, straddling the divide between computer programming and design. The field moves at a breakneck pace and never stops changing. New languages, frameworks, technologies, and trends never leave a dull moment for the web developer, as there is always something new to learn.

In this guided, two-part activity, participants will be shown how a simple modern web application is assembled using a component-based framework. They will also learn how to add interactivity to web applications and customize the design. They will also learn how to pull data from a web API and update the application's state to create a seamless user experience.

Part 1: HTML and CSS

HTML and CSS are fundamental parts of any web application. HTML defines the content and structure, and CSS provides the style.

HTML, which stands for HyperText Markup Language, consists of tags (<></>), which describe the structure of the content. Different tags can be used to create various structures.

CSS, which stands for Cascading Style Sheets, describes how and where elements should be placed on the page. With CSS, developers can define an element's color, size, shape, and location. CSS is commonly applied to HTML using the class attribute.

Part 3: Start Game

All games, even very simple ones, rely on data. When starting a new game, it is essential to know what data is needed and what initial values that data should have.

For a memory game, we will need a deck of cards that has been "shuffled." We will also need to keep track of flipped cards, matched cards, and the number of moves the player has taken. Finally, we must know if the game is still going or active.

Info

Part 4: Game Loop

A game loop is a series of actions the player takes and the responses to those actions. The series repeats until a determined "end" is reached.

In Memory, the game loop starts with the player clicking on two cards. Then, the computer increases the moves property and checks if the cards are a match. If they are, they are marked as a match; if not, they are flipped back over. The loop will continue until all matches have been found.

Step 3

When the player has flipped two cards, increase the value of the moves property by 1. Use an if statement and the length of the flippedCards to check if two cards have been flipped, and then use the increment operator (++) to increase moves.

JS
flipCard: function (card) {
  card.flipped = true
  this.flippedCards.push(card)

  if (this.flippedCards.length === 2) {
    this.moves++
  }
}

Step 4

Prevent the player from clicking on cards that have already been "flipped" or "matched," or if two cards have already been flipped. Add an if statement at the beginning of the flippedCard method to check if the card's flipped or matched properties are true or if the length property of flippedCards is equal to 2. If so, exit the method.

JS
flipCard: function (card) {
  if (card.flipped || card.matched || this.flippedCards.length === 2) {
    return
  }

  card.flipped = true
  this.flippedCards.push(card)

  if (this.flippedCards.length === 2) {
    this.moves++
  }
}

Step 5

After increasing the value of moves, invoke the checkCards method. The checkCards method will check if the cards are a match and perform the appropriate actions if they are.

JS

Part 5: End Game

When a game has concluded its objective, it enters the end game. This may happen when a player runs out of lives or defeats the final boss. In our game, it will occur when the user has found all of the matches.

End games usually include a closing sequence and an opportunity for the player to play again.

Tips

For Part 5, we will be making changes to the "HTML", "CSS" and "JS" boxes.

Step 1

After invoking the checkCards method, check if the player has reached the end of the game by comparing the length of the matchedCards array to the cards array. Then set the active property to false.

JS
flipCard: function (card) {
  if (card.flipped || card.matched || this.flippedCards.length === 2) {
    return
  }

  card.flipped = true
  this.flippedCards.push(card)

  if (this.flippedCards.length === 2) {
    this.moves++
    this.checkCards()

    if (this.matchedCards.length === this.cards.length) {
      this.active = false
    }
  }
}

Step 2

Let the player play again by clicking the "Play Again" button. Add @click directive to the "Play Again" button and point it to the startGame method.

HTML
<div class="popup" :class="{show: !active}">
  <button class="btn btn-lg btn-primary" @click="startGame">Play Again</button>
</div>

Props are a way of sending data to the component to alter what the component displays. Props are passed to the component using attributes.

Part 4: Event Handling

Interactive web applications require event handling. Events are actions that occur in the browser. These actions may be triggered by the user or by the browser itself.

In React, events are handled using special JSX attributes like onClick or onChange. These attributes are linked using a function, which is invoked when the event is triggered.

When an event occurs on a child component that needs to invoke a function in the parent component, the parent component will pass the function as a prop.
