Web storage is a more modern approach to client-side storage, introduced with HTML5. Unlike cookies, web storage does not automatically send data to the server with each request. However, it has a much larger storage capacity than cookies, between 5 and 10MB. It provides two different storage mechanisms: **local storage** and **session storage**. 

## Local Storage vs. Session Storage

While local storage and session storage are very similar in implementation and usage, they differ in the persistence of the stored data. Local storage stores data with no expiration date and will remain even after the browser has been closed. On the other hand, session storage only stores data for the duration of the page session. Once the page or browser is closed, the data is deleted.

## Using Local Storage

The [`localStorage`](https://developer.mozilla.org/en-US/docs/Web/API/Window/localStorage) object in JavaScript allows access to the browser's localStorage, which is a way of saving data across browser sessions. The data to localStorage is domain-specific.  

> #### NOTE
> The [`sessionStorage`](https://developer.mozilla.org/en-US/docs/Web/API/Window/sessionStorage) object and the browser's sessionStorage are similar to localStorage except that stored data will be cleared when the page session ends.

### Storing to Local Storage
The `setItem()` function is used to store new data items to local storage. The function takes two arguments, the first being the item's key and the second being the item's value. Both the key and value must be a string. 

```js
localStorage.setItem('name', 'Michael')
localStorage.setItem('title', 'Professor')
```

### Retrieving Local Storage Items
The `getItem()` function is used to retrieve data items stored in local storage. The function takes a single argument, the key of the desired item. 

```js
localStorage.getItem('name') // Michael
localStorage.getItem('title') // Professor
```

### Deleting Local Storage
The `removeItem()` function is used to remove an item from local storage. It takes a single argument, the key of the desired item. 

The `clear()` function is used to clear all items stored in local storage for the current domain.

```js
localStorage.removeItem('name')
localStorage.clear()
```

## Storing Objects to Local Storage
Local storage can only store strings. This can be quite limiting for web applications that must store and keep track of more complex data. 

A workaround for this is to store an entire object or array to a data item as a string using the [`JSON.stringify()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/JSON/stringify) function. This function takes an object or an array and returns it as a string. This string can then be stored in local storage.

```js
const string = JSON.stringify({ name: 'Michael', title: 'Professor' })
// "{"name":"Michael","title":"Professor"}"

localStorage.setItem('data', string)
```

When it comes to retrieving the data item, the [`JSON.parse()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/JSON/parse) function can be used. This function takes a previously stringified object or array and returns a fully functional object or array. 

```js
const data = JSON.parse(localStorage.getItem('data'))

data.name // Michael
```
